# Network Optimization and Capacity Planning

## Overview

The final stage of the project focuses on network dimensioning and capacity optimization. After collecting and processing traffic measurements, the resulting traffic matrices are used as the input of a Linear Programming (LP) model. This model determines the optimal bandwidth required for each backbone link while minimizing the overall deployment cost.

Instead of relying on manual estimations, the optimization process provides an objective and mathematically validated solution that satisfies all traffic demands while respecting network capacity constraints.

---

## Objectives

The optimization process was designed to achieve the following objectives:

- Minimize the overall infrastructure cost (CAPEX and OPEX).
- Determine the optimal bandwidth allocation for every backbone link.
- Prevent congestion by satisfying all traffic demands.
- Select efficient routing paths across the network topology.
- Generate a scalable network design suitable for future traffic growth.

---

## Optimization Model

The network is modeled as a Mixed Integer Linear Programming (MILP) problem solved with the GNU Linear Programming Kit (GLPK).

The optimization simultaneously evaluates every possible routing alternative and determines the minimum-cost solution while satisfying all engineering constraints.

---

## Decision Variables

The optimization model uses two groups of decision variables.

### Traffic Variables

`xdp`

Continuous variables representing the amount of traffic assigned to a specific routing path between an origin and destination pair.

### Capacity Variables

`ye`

Integer variables representing the capacity assigned to each physical backbone link.

---

## Optimization Constraints

Several constraints guarantee that every solution is technically feasible.

### Traffic Demand

The total traffic assigned to all available paths must exactly satisfy the measured traffic demand.

\[
\sum_p x_{dp}=h_d
\]

---

### Link Capacity

The total traffic traversing every physical link cannot exceed its installed capacity.

\[
\sum_d\sum_p \delta_{edp}x_{dp}\le M_ey_e
\]

---

## Objective Function

The optimization minimizes the total deployment cost by combining both investment costs (CAPEX) and operational costs (OPEX).

\[
\min \sum_{e=1}^{E}(CAPEX_e+OPEX_e)\,y_e
\]

The solution determines the minimum amount of bandwidth that satisfies every network constraint.

---

## GNU Octave Environment

The optimization model is implemented in GNU Octave.

Octave imports the processed traffic matrices generated by the Python scripts, constructs the optimization matrices and calls the GLPK solver to compute the optimal solution.

---

## GLPK Solver

The GNU Linear Programming Kit (GLPK) is used to solve the optimization problem.

The solver applies linear programming algorithms to determine:

- optimal traffic allocation
- required link capacities
- minimum deployment cost

while satisfying every network constraint.

---

## Input Data

The optimization receives two different traffic matrices produced during the previous analysis stage.

### Average Traffic Matrix

Represents the average network load obtained after filtering abnormal samples.

This matrix is mainly used to evaluate typical operating conditions.

### 99th Percentile Traffic Matrix

Represents sustained peak traffic while ignoring isolated traffic bursts.

Using the 99th percentile prevents unnecessary over-dimensioning while maintaining a high quality of service.

---

## Capacity Planning Strategy

Before running the optimization, the measured traffic values are standardized and prepared.

The following engineering assumptions are applied:

- Traffic values are converted into bits per second (bps).
- Future traffic growth is considered using a scaling factor.
- Different capacity blocks are assigned depending on the type of backbone link.
- Regional and intercontinental links have different deployment costs.

This methodology produces a realistic long-term capacity planning model.

---

## Output Results

After the optimization completes, the model produces:

- Optimal traffic allocation for every communication path.
- Required capacity for every physical backbone link.
- Minimum deployment cost.
- Capacity blocks required for each link.
- Optimal routing distribution.

These outputs provide the final network design used for deployment planning.

---

## Design Decisions

Several engineering decisions were adopted during the optimization process.

- Linear Programming was selected because it guarantees globally optimal solutions.
- Mixed Integer Programming models real telecommunications capacity blocks.
- Traffic growth is considered during the planning stage to improve scalability.
- The 99th percentile is preferred over maximum traffic values to avoid over-provisioning.
- Cost parameters differentiate regional and intercontinental backbone links.

---

## Benefits of the Optimization Model

The optimization process provides several important advantages:

- Minimizes deployment costs.
- Prevents network congestion.
- Maximizes infrastructure utilization.
- Produces objective and reproducible results.
- Supports future network expansion.
- Provides an engineering-based capacity planning methodology.

## Summary

The optimization stage transforms traffic measurements into a complete network capacity planning solution.

Using traffic matrices generated from NetFlow analysis, GNU Octave and the GLPK solver determine the optimal bandwidth allocation for every backbone link while minimizing CAPEX and OPEX. This mathematical approach produces a scalable, cost-efficient and technically validated network design suitable for real-world telecommunications infrastructures.
