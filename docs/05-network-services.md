# Network Services

## Overview

Several network services were deployed to support the operation of the laboratory infrastructure. These services provided secure remote administration, local software installation, traffic generation, and NetFlow data collection for subsequent network analysis and optimization.

---

## SSH Remote Administration

SSH was used to remotely access and configure the MikroTik routers through the dedicated management network.

The management interfaces are isolated from the production network, allowing administrators to configure devices without affecting user traffic.

### Management Network

- Private addressing: `192.168.X.X`
- Remote administration of all MikroTik routers
- Out-of-band (OOB) management
- Secure command-line access

### Example Command

```bash
ssh -o "StrictHostKeyChecking=no" admin@192.168.3.11
```

## Local Repository (Eihal)

The laboratory operates without Internet access. To install software packages, a local repository server named **Eihal** is used.

The server hosts Ubuntu repositories and installation scripts, allowing Linux hosts to install networking tools without external connectivity.

### Server Information

| Parameter | Value |
|-----------|-------|
| Server Name | Eihal |
| IP Address | 192.168.33.21 |
| Purpose | Local software repository |

### Repository Scripts

```bash
sh repository_ubuntu_precise.sh

sudo sh repository_ubuntu_bionic.sh
```

## iPerf

iperf is used to generate traffic between hosts in order to validate routing, measure throughput, and create realistic traffic matrices for network analysis.

### Server Mode

Each Linux host starts one or more background iPerf server processes listening on dedicated ports.

Example:

```bash
iperf -s -D -p 5001
```

### Client Mode

Client hosts establish simultaneous connections with remote hosts for one hour.

Example:

```bash
iperf -c 33.3.3.5 -p 5005 -t 3600 &
```

### Ports

- 5001
- 5002
- 5003
- 5004
- 5005
- 5006
- 5007

### Related Scripts

- `SERVIDOR.sh`
- `CLIENTE.sh`
- `scriptservidorIPERF.sh`
- `scriptclienteIPERF.sh`

## NetFlow Services

Traffic monitoring is implemented using MikroTik Traffic Flow together with Linux collection tools.

### nfcapd

Receives NetFlow Version 9 packets exported by the MikroTik routers and stores them as binary flow files.

Example:

```bash
nfcapd -w -D -p 5022 -l /home/operador/Escritorio/nfcapd -t 60
```

### nfdump

Reads the binary NetFlow files generated by nfcapd and exports traffic information into CSV format for later processing.

Example:

```bash
nfdump -r nfcapd.20251210110001 \
-o csv \
-A srcip,dstip \
'src ip 33.3.1.1'
```

### NetFlow Configuration

| Parameter | Value |
|-----------|-------|
| Export Protocol | NetFlow v9 |
| Collector | nfcapd |
| Processing Tool | nfdump |
| UDP Port | 5022 |
| File Rotation | 60 seconds |

### Related Scripts

- `scriptNetflow.sh`
- `configrouterscogerinfo.sh`
- `nfcapd.sh`
- `nfdump.sh`

## Summary

The deployed network services provide all the functionality required to operate the laboratory infrastructure.

They enable secure router administration, software installation in an isolated environment, traffic generation, NetFlow data collection, and automated processing of network measurements. Together, these services support the traffic engineering and network optimization tasks performed throughout the project.
