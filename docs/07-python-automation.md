# Python Automation

## Overview

Python was used to automate the processing and analysis of the traffic data collected through NetFlow. Instead of manually analyzing CSV files, the automation pipeline imports the captured traffic, cleans invalid samples, computes statistical metrics, and generates the traffic matrices required for network dimensioning and optimization.

The complete analysis is implemented in the main script `salsa.py`, which transforms raw NetFlow exports into meaningful engineering metrics.

---

## Objectives

The Python automation was developed to achieve the following objectives:

- Import NetFlow CSV datasets automatically.
- Extract traffic measurements from multiple network flows.
- Remove invalid samples and traffic anomalies.
- Compute average traffic and 99th percentile values.
- Generate traffic matrices for network planning.
- Prepare the processed data for optimization with GNU Octave and GLPK.

## Script Structure

The analysis is implemented in the main script `salsa.py`, which includes several functions responsible for processing the captured traffic data.

### CSV Parsing

Reads the CSV files generated by **nfdump**, extracts the traffic (Bytes) column, and converts it into numerical arrays.

### Traffic Visualization

Generates line graphs using Matplotlib to visualize traffic behaviour and detect abnormal values.

### Data Filtering

Removes startup periods, invalid measurements, and transient traffic spikes by applying lower and upper thresholds.

### Statistical Processing

For every cleaned traffic series, the script computes:

- Average traffic
- 99th percentile
- Processed traffic vector

---

## Processing Workflow

The automated processing pipeline follows these steps:

1. Read the CSV files generated from NetFlow captures.
2. Extract the traffic column.
3. Plot each traffic series.
4. Remove anomalous samples.
5. Calculate the average traffic.
6. Multiply the average value by two to model a maximum link utilization of 50%.
7. Calculate the 99th percentile.
8. Generate the final traffic matrices.

---

## Traffic Visualization

Traffic graphs are used to identify abnormal measurements before statistical analysis.

Typical anomalies include:

- Initial periods without traffic before the experiment begins.
- Short traffic bursts generated during startup.
- Invalid or inconsistent measurements.

After filtering these samples, only steady-state traffic remains, producing more reliable engineering results.

---

## Traffic Metrics

Two complementary traffic metrics are calculated.

### Average Traffic

The average traffic is computed using the arithmetic mean of the cleaned data series.

To design the network assuming a maximum utilization of **50%**, the average value is multiplied by two. This provides a conservative estimate of the required link capacity.

### 99th Percentile

The 99th percentile represents sustained peak traffic.

Unlike the absolute maximum, it ignores isolated bursts while still representing realistic peak network demand. This metric is commonly used for traffic engineering and capacity planning.

---

## Design Decisions

The following implementation decisions were adopted:

- **NumPy** was selected for efficient numerical processing.
- **Matplotlib** was used to visualize traffic series and detect anomalies.
- Manual threshold filtering was preferred over automatic outlier removal to maintain full control over the cleaning process.
- The **99th percentile** was chosen instead of the absolute maximum to avoid unnecessary network over-dimensioning.
- Average traffic values were multiplied by two to model a network operating at a maximum utilization of 50%.

## Summary

The Python automation layer converts raw NetFlow data into meaningful traffic information required for network engineering.

Using Python, NumPy and Matplotlib, the project automatically imports captured traffic, filters anomalous samples, computes statistical metrics, and generates the traffic matrices used during the optimization phase. This automated workflow significantly reduces manual effort while improving the accuracy, reproducibility, and efficiency of the network analysis.
